My goal for the day was to experiment and tinker with all the elements on the Twitter Clone.  I wanted to understand how each language interacted with each other.  How are elements arrayed?  What are the limitations of inserting CSS into Ruby code?  What can I do with this Boostrap grid system?  Can I create a new Tweet without the 'New' Tweet page?  Here is a screenshot of what I have at the end of the day:

Per one of my questions from the previous day, I wanted to see if I could inject CSS and Javascript into the Ruby code that created forms.  After a bit of research and searching I found that it was indeed possible. By typing,

{%highlight ruby%}<%= f.text_area :text, class: 'form-control status-box' %>{%endhighlight%}

I was able to assign the classes 'form_control' and 'status-box' to the text_area.  It turns out the 'form_for' Ruby function is simply a helper that generates an HTML form tag, action and method.  Since it is converted to HTML it is a small step to apply JavaScript to those elements.  Here is the guide I used where I found this information, Working with JavaScript in Rails

There were some cool Bootstrap features that I was able to implement.  Text_areas and buttons, when assigned a 'form-group' class, include a measure of client-side validation through Bootstrap.  To incorporate this validation, I disabled the 'Post' button and changed the text_area to red when posts reach longer than 140 characters.  The button with class 'btn-post' is assigned a 'disabled' class and the 'status-box' text_area is assigned a 'has-error' class.
