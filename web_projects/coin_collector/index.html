Development

If you are interested in learning about the development of Coin Collector, I wrote a four-part series on my blog describing various aspects of the game.

  • Part 1 - Architecture: This post describes how I constructed the underlying structure of Coin Collector. It goes into detail about why gameplay loops and browser architecture are fundamentally different and how I went about solving this problem.
  • Part 2 - Terrain Generation: I wanted this game to have procedurally generated terrain that would create a unique landscape for every playthrough. This post talks about how the landscape is constructed and what restrictions are in place to create a challenging yet traversable landscape.
  • Part 3 - Collision Detection: What would this game be without a character jumping around and interacting with the environment? I take some time here to explain how the character and environment are represented within the code and how the game calculates collisions between the two.
  • Part 4 - Animations: Animations provide those final details that make a game come to life! This post delves into hit boxes vs. animation images and how the game state determines which animation frame needs to play at that specific moment in time.
